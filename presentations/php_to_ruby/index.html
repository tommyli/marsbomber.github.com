Syntax Difference in Ruby

Variables have no leading $

fruit = "banana"

Statements have no tailing ;

fruit = "banana"

Parentheses are optional.

s = String.new "Banana"
s.concat " Boat"

Object Oriented - Ruby

Everything is an object. That includes booleans, number and even nil (null in php). This means more concise syntax. Always use method calls on objects, no function calls.

> true.class  #=> TrueClass
> 7.class     #=> Fixnum
> nil.class   #=> NilClass

This allows us to do some cool things.

> 2.times {|n| puts "#{n}: Ruby is cool"}
0: Ruby is cool
1: Ruby is cool

Object Oriented - PHP

PHP 5 introduced OO. The language is bloated with concepts from both procedural and OO paradigm. This leads to syntactic inconsistencies.

> $a = new stdClass(); echo is_object($a);  #=> 1
> $b = true;           echo is_object($b);  #=> ...
